Week 2 Research/Discussion

1. Less obvious perks of using 'live' data resources include logins and payments. How could Instagram be used in these two scenarios?

So long as I understood the GoTo Training correctly we were supposed to put Instagram in place of Twitter. Instagram could use these two scenarios by say using Facebook for logins and PayPal for payments. This one prevents the user from having to have multiple different logins if they are using their facebook account to login with. And PayPal is a trusted source that the user can use to make purchases.

2. Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

Because it allows us to request data from a server in a different domain, instead of the information having to be on the same domain.

3. Based on JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

MapQuest Geocoding may be one data resource that may pair well with the gps native device feature for my app. Say my user wants to mark a location of a store they were at to the list item they added, this may be a great feature to work for it.

4. Please describe any design and functionality intentions you may have for your Demo app components this month.

So I decided to incorporate my app from our previous class for this month. It's a list app, not necessarily a "TO Do" list app but just a list, either groceries, supplies, anything really you may need to make a list of and where to go for that list. I also feel that using the barcode camera feature would be good for this app, If I can make ti work the where the user if out at the store sees something they want to add to their list, they can use the barcode scanner to add that item to the list and hopefully will add of the information, price, etc to it also.

5. Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

Some past lessons learned that I use now, would definitely be planning out my workflow, and time management for that. Coming up with the idea for the app, then setting up a design workflow of how I want that app to look. Then going to do some research on similar apps, and then coding.
